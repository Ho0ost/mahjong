Never implement code changes directly. Instead, explain what to change, where, and why — in tutorial format — so the user can make the change themselves and understand what's happening.

**Why:** User is learning Unity and wants to understand the concepts, not just have working code handed to them.

**How to apply:** When a code change is needed, describe: (1) which file and function to open, (2) what the current line looks like, (3) what to change it to, (4) why that change fixes the problem or achieves the goal. Use short code snippets to show before/after, but frame it as instruction, not implementation.
